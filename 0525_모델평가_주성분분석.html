모델 평가¶

MSE¶

- 모델 예측값과 실제값 간의 오차 제곱합의 평균

RMSE¶

- 평균제곱오차의 제곱근  
- MSE에 루트를 취한 것

MAE¶

- 모델 예측값과 실제값 간의 오차 절대값 합의 평균  
- 이상치에 취약

MAPE¶

- 평균오차를 비율로 나타낸 것
- MSE, RMSE의 단점(scale에 영향)을 보완

# RMSE: root mean squared error 함수
def root_mean_squared_error(y_true, y_pred):
    return np.sqrt(mean_squared_error(y_true, y_pred))

# MAPE: mean absolute percentage error 함수
def mean_absolute_percentage_error(y_true, y_pred):
    return np.mean(np.abs((y_true - y_pred) / y_true)) * 100

MSE, RMSE, MAE, MAPE의 변화 추이는 각 모델마다 비슷하다고 볼 수 있다.

  1. 회귀분석
  2. 랜덤 포레스트, 그래디언트 부스팅
  3. 의사결정나무

순으로 성능이 좋다. 회귀분석은 성능이 좋지 않은 변수를 하나씩 제거하기 때문에 가장 좋은 성능을 보였다고 생각한다. 랜덤 포레스트와 그래디언트 부스팅은 여러 트리를 만들고 이를 종합하여 모델을 생성하기 때문에 그 다음으로 좋은 성능을 보였다. 마지막으로 의사결정나무는 train 데이터를 많이 사용하기 때문에 test 데이터에 대해 성능이 좋지 않아 가장 성능이 나쁘게 나온 것 같다.

주성분 분석¶

  • 정의: 변수 간의 관계(공분산, 상관)를 기반으로 정보 손실을 최소화하면서 주요 성분을 확인하고 차원을 축소하는 기법
  • 주요 성분 확인
  • 차원 축소: 변수를 축약하여 정보를 쉽게 표현
  • 유의 사항: 변수의 scale이 유사해야 한다.
    • scale이 유사하면 공분산 행렬
    • scale 차이가 크면 상관계수 이용
